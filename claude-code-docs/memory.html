記憶・設定・權限

CLAUDE.md 記憶系統、自動記憶、設定檔層級、權限模式、互動模式、CLI 參考——你和 Claude 之間的契約。

Claude 如何記住你的專案

使用 CLAUDE.md 檔案為 Claude 提供持久指令，並讓 Claude 透過自動記憶（Auto Memory）功能自動累積學習內容。

每個 Claude Code 會話都從一個全新的上下文視窗開始。兩種機制可以跨會話傳遞知識：

  • CLAUDE.md 檔案：你編寫的指令，為 Claude 提供持久上下文
  • 自動記憶（Auto Memory）：Claude 根據你的更正和偏好自己編寫的筆記

CLAUDE.md 與自動記憶

Claude Code 有兩個互補的記憶系統。兩者都在每次對話開始時載入。Claude 將它們視為上下文，而不是強制配置。你的指令越具體和簡潔，Claude 遵循它們的一致性就越高。

項目 CLAUDE.md 檔案 自動記憶
誰編寫 你 Claude
包含內容 指令和規則 學習和模式
範圍 專案、使用者或組織 每個工作樹（worktree）
載入到 每個會話 每個會話（前 200 行）
用於 編碼標準、工作流、專案架構 構建命令、除錯見解、Claude 發現的偏好

當你想指導 Claude 的行為時，使用 CLAUDE.md 檔案。自動記憶讓 Claude 從你的更正中學習，無需手動操作。

CLAUDE.md 檔案

CLAUDE.md 檔案是 markdown 檔案，為專案、你的個人工作流或整個組織為 Claude 提供持久指令。你用純文字編寫這些檔案；Claude 在每個會話開始時讀取它們。

選擇 CLAUDE.md 檔案的位置

CLAUDE.md 檔案可以位於多個位置，每個位置有不同的範圍。更具體的位置優先於更廣泛的位置。

範圍 位置 目的 用例示例 共享對象
專案指令 ./CLAUDE.md 或 ./.claude/CLAUDE.md 專案的團隊共享指令 專案架構、編碼標準、常見工作流 透過原始碼控制的團隊成員
使用者指令 ~/.claude/CLAUDE.md 所有專案的個人偏好 程式碼樣式偏好、個人工具快捷方式 僅你（所有專案）

工作目錄上方目錄層次結構中的 CLAUDE.md 檔案在啟動時完整載入。子目錄中的 CLAUDE.md 檔案在 Claude 讀取這些目錄中的檔案時按需載入。

設定專案 CLAUDE.md

專案 CLAUDE.md 可以存儲在 ./CLAUDE.md 或 ./.claude/CLAUDE.md 中。創建此檔案並添加適用於在專案上工作的任何人的指令：構建和測試命令、編碼標準、架構決策、命名約定和常見工作流。這些指令透過版本控制與你的團隊共享。

執行 /init 自動生成起始 CLAUDE.md。Claude 分析你的程式碼庫並創建一個包含構建命令、測試指令和它發現的專案約定的檔案。如果 CLAUDE.md 已存在，/init 會建議改進而不是覆蓋它。

編寫有效的指令

CLAUDE.md 檔案在每個會話開始時載入到上下文視窗中。因為它們是上下文而不是強制配置，你編寫指令的方式會影響 Claude 遵循它們的可靠性。具體、簡潔、結構良好的指令效果最好。

  • 大小：每個 CLAUDE.md 檔案目標在 200 行以下。較長的檔案消耗更多上下文並降低遵守度。
  • 結構：使用 markdown 標題和項目符號來分組相關指令。有組織的部分比密集段落更容易遵循。
  • 具體性：編寫具體到足以驗證的指令。例如：「使用 2 空格縮排」而不是「正確格式化程式碼」
  • 一致性：定期審查你的 CLAUDE.md 檔案以刪除過時或衝突的指令。

匯入其他檔案

CLAUDE.md 檔案可以使用 @path/to/import 語法匯入其他檔案。匯入的檔案在啟動時展開並載入到上下文中。

允許相對路徑和絕對路徑。相對路徑相對於包含匯入的檔案解析。匯入的檔案可以遞迴匯入其他檔案，最大深度為五跳。

有關專案概述，請參閱 @README，有關此專案的可用 npm 命令，請參閱 @package.json。

# 其他指令
- git 工作流 @docs/git-instructions.md

對於你不想簽入的個人偏好，從你的主目錄匯入檔案：

# 個人偏好
- @~/.claude/my-project-instructions.md
Claude Code 第一次在專案中遇到外部匯入時，它會顯示一個批准對話框。如果你拒絕，匯入保持停用狀態。

AGENTS.md

Claude Code 讀取 CLAUDE.md，而不是 AGENTS.md。如果你的存儲庫已經為其他編碼代理使用 AGENTS.md，創建一個匯入它的 CLAUDE.md，這樣兩個工具都可以讀取相同的指令：

@AGENTS.md

## Claude Code

對 `src/billing/` 下的更改使用 plan mode。

CLAUDE.md 檔案如何載入

Claude Code 透過從當前工作目錄向上遍歷目錄樹來讀取 CLAUDE.md 檔案。如果你在 foo/bar/ 中執行 Claude Code，它會從 foo/bar/CLAUDE.md 和 foo/CLAUDE.md 載入指令。

Claude 還在當前工作目錄下的子目錄中發現 CLAUDE.md 檔案。它們不是在啟動時載入，而是在 Claude 讀取這些子目錄中的檔案時包含。

區塊級 HTML 註解（<!-- maintainer notes -->）在 CLAUDE.md 檔案中在內容注入到 Claude 的上下文之前被剝離。

使用 `.claude/rules/` 組織規則

對於較大的專案，你可以使用 .claude/rules/ 目錄將指令組織到多個檔案中。規則也可以範圍限定到特定檔案路徑，因此它們僅在 Claude 處理匹配檔案時載入到上下文中。

規則在每個會話或打開匹配檔案時載入到上下文中。對於不需要始終在上下文中的特定任務指令，改用 skills。

設定規則

在你的專案的 .claude/rules/ 目錄中放置 markdown 檔案。每個檔案應涵蓋一個主題，具有描述性檔名，如 testing.md 或 api-design.md。

your-project/
├── .claude/
│   ├── CLAUDE.md           # 主專案指令
│   └── rules/
│       ├── code-style.md   # 程式碼樣式指南
│       ├── testing.md      # 測試約定
│       └── security.md     # 安全要求

特定路徑的規則

規則可以使用帶有 paths 欄位的 YAML frontmatter 範圍限定到特定檔案：

---
paths:
  - "src/api/**/*.ts"
---

# API 開發規則

- 所有 API 端點必須包括輸入驗證
- 使用標準錯誤回應格式
- 包括 OpenAPI 文件註解

在 paths 欄位中使用 glob 模式按副檔名、目錄或任何組合匹配檔案：

模式 匹配
**/*.ts 任何目錄中的所有 TypeScript 檔案
src/**/* src/ 目錄下的所有檔案
*.md 專案根目錄中的 Markdown 檔案
src/components/*.tsx 特定目錄中的 React 元件

自動記憶

自動記憶讓 Claude 跨會話累積知識，無需你編寫任何內容。Claude 在工作時為自己儲存筆記：構建命令、除錯見解、架構筆記、程式碼樣式偏好和工作流習慣。

自動記憶需要 Claude Code v2.1.59 或更高版本。使用 claude --version 檢查你的版本。

啟用或停用自動記憶

自動記憶預設開啟。要切換它，在會話中打開 /memory 並使用自動記憶切換，或在你的專案設定中設定 autoMemoryEnabled：

{
  "autoMemoryEnabled": false
}

要透過環境變數停用自動記憶，設定 CLAUDE_CODE_DISABLE_AUTO_MEMORY=1。

儲存位置

每個專案在 ~/.claude/projects/<project>/memory/ 獲得自己的記憶目錄。<project> 路徑來自 git 存儲庫。

要將自動記憶儲存在不同位置，在你的使用者或本地設定中設定 autoMemoryDirectory：

{
  "autoMemoryDirectory": "~/my-custom-memory-dir"
}

目錄包含一個 MEMORY.md 入口點和可選的主題檔案。MEMORY.md 的前 200 行在每次對話開始時載入。

審計和編輯你的記憶

自動記憶檔案是純 markdown，你可以隨時編輯或刪除。執行 /memory 從會話中瀏覽和打開記憶檔案。

故障排除記憶問題

Claude 不遵循我的 CLAUDE.md

CLAUDE.md 內容作為使用者訊息傳遞。Claude 讀取它並嘗試遵循它，但沒有嚴格遵守的保證。

要除錯：

  • 執行 /memory 驗證你的 CLAUDE.md 檔案被載入
  • 檢查相關 CLAUDE.md 是否在為你的會話載入的位置
  • 使指令更具體
  • 查找跨 CLAUDE.md 檔案的衝突指令

我的 CLAUDE.md 太大了

超過 200 行的檔案消耗更多上下文並可能降低遵守度。將詳細內容移到使用 @path 匯入引用的單獨檔案中，或將你的指令分割到 .claude/rules/ 檔案中。

Claude Code 設定

使用全域和專案級設定以及環境變數配置 Claude Code。你可以在使用互動式 REPL 時執行 /config 命令來配置 Claude Code。

配置作用域

Claude Code 使用作用域系統來確定配置應用的位置以及與誰共享。

作用域 位置 影響範圍 與團隊共享？
User ~/.claude/ 目錄 你，跨所有專案 否
Project 存儲庫中的 .claude/ 此存儲庫上的所有協作者 是（提交到 git）
Local .claude/settings.local.json 你，僅在此存儲庫中 否（gitignored）

設定檔案

settings.json 檔案是透過分層設定配置 Claude Code 的官方機制：

  • 使用者設定在 ~/.claude/settings.json 中定義，適用於所有專案。
  • 專案設定保存在你的專案目錄中：
    • .claude/settings.json 用於簽入原始碼管理並與團隊共享的設定
    • .claude/settings.local.json 用於未簽入的設定，適用於個人偏好和實驗
{
  "$schema": "https://json.schemastore.org/claude-code-settings.json",
  "permissions": {
    "allow": [
      "Bash(npm run lint)",
      "Bash(npm run test *)",
      "Read(~/.zshrc)"
    ],
    "deny": [
      "Bash(curl *)",
      "Read(./.env)",
      "Read(./.env.*)",
      "Read(./secrets/**)"
    ]
  },
  "env": {
    "CLAUDE_CODE_ENABLE_TELEMETRY": "1",
    "OTEL_METRICS_EXPORTER": "otlp"
  }
}

排除敏感檔案

要防止 Claude Code 存取包含敏感資訊的檔案，請在 .claude/settings.json 中使用 permissions.deny 設定：

{
  "permissions": {
    "deny": [
      "Read(./.env)",
      "Read(./.env.*)",
      "Read(./secrets/**)",
      "Read(./config/credentials.json)"
    ]
  }
}

配置權限

透過細粒度權限規則、模式和託管策略來控制 Claude Code 可以存取和執行的操作。

權限系統

工具類型 示例 需要批准
唯讀 檔案讀取、Grep 否
Bash 命令 Shell 執行 是
檔案修改 Edit/Write 檔案 是

管理權限

你可以使用 /permissions 查看和管理 Claude Code 的工具權限。

  • Allow 規則讓 Claude Code 使用指定的工具而無需手動批准
  • Ask 規則在 Claude Code 嘗試使用指定工具時提示確認
  • Deny 規則防止 Claude Code 使用指定的工具

規則按順序評估：deny → ask → allow。第一個匹配的規則獲勝。

權限模式

模式 說明 最適合
default 標準行為：在首次使用每個工具時提示權限 入門、敏感工作
acceptEdits 自動接受會話的檔案編輯權限 迭代你正在審查的程式碼
plan Plan Mode：Claude 可以分析但不能修改檔案或執行命令 探索程式碼庫、規劃重構
auto 自動批准工具呼叫，並進行後台安全檢查（研究預覽階段） 長時間執行的任務、減少提示疲勞
dontAsk 自動拒絕工具，除非透過規則預先批准 鎖定環境
bypassPermissions 跳過權限提示，除了對受保護目錄的寫入 僅隔離容器和 VM
bypassPermissions 模式跳過權限提示。僅在隔離環境（如容器或 VM）中使用此模式。

權限規則語法

權限規則遵循格式 Tool 或 Tool(specifier)。

匹配所有使用：

  • Bash — 匹配所有 Bash 命令
  • WebFetch — 匹配所有網路擷取請求
  • Read — 匹配所有檔案讀取

細粒度控制：

  • Bash(npm run build) — 匹配確切的命令
  • Read(./.env) — 匹配讀取當前目錄中的 .env 檔案
  • WebFetch(domain:example.com) — 匹配對 example.com 的擷取請求

Bash 規則支援帶有 * 的 glob 模式：

{
  "permissions": {
    "allow": [
      "Bash(npm run *)",
      "Bash(git commit *)",
      "Bash(* --version)"
    ],
    "deny": [
      "Bash(git push *)"
    ]
  }
}
Read 和 Edit deny 規則適用於 Claude 的內建檔案工具，不適用於 Bash 子程序。Read(./.env) deny 規則阻止 Read 工具，但不會阻止 Bash 中的 cat .env。要獲得 OS 級別的強制執行，請啟用沙箱。

切換權限模式

在會話期間：按 Shift+Tab 循環切換 default → acceptEdits → plan → auto。當前模式顯示在狀態列中。

啟動時：

claude --permission-mode plan

作為預設設定：

{
  "permissions": {
    "defaultMode": "acceptEdits"
  }
}

使用 Plan Mode 在編輯前進行分析

Plan Mode 告訴 Claude 研究並提議更改而不進行更改。Claude 讀取檔案、執行 shell 命令進行探索、提出澄清問題並編寫計劃檔案，但不編輯你的原始碼。

何時使用 Plan Mode：

  • 多步驟實作：當功能需要跨多個檔案的編輯時
  • 程式碼探索：當你想在更改任何內容前研究程式碼庫時
  • 互動式開發：當你想與 Claude 迭代方向時
claude --permission-mode plan

提示建議

--print, -p 列印回應而不進入互動模式
--resume, -r 按 ID 或名稱恢復特定會話
--version, -v 輸出版本號
--worktree, -w 在隔離的 git worktree 中啟動 Claude

常見工作流

使用 Claude Code 探索程式碼庫、修復錯誤、重構、測試和其他日常任務的分步指南。

理解新的程式碼庫

快速取得程式碼庫概覽：

  1. 導覽到專案根目錄：cd /path/to/project
  2. 啟動 Claude Code：claude
  3. 請求高級概覽：give me an overview of this codebase
  4. 深入了解特定元件：
    • explain the main architecture patterns used here
    • what are the key data models?
    • how is authentication handled?

修復錯誤

  1. 與 Claude 分享錯誤：I'm seeing an error when I run npm test
  2. 請求修復建議：suggest a few ways to fix the @ts-ignore in user.ts
  3. 應用修復：update user.ts to add the null check you suggested

重構程式碼

  1. 識別用於重構的遺留程式碼：find deprecated API usage in our codebase
  2. 取得重構建議：suggest how to refactor utils.js to use modern JavaScript features
  3. 安全地應用更改：refactor utils.js to use ES2024 features while maintaining the same behavior
  4. 驗證重構：run tests for the refactored code

使用 Plan Mode 進行安全的程式碼分析

Plan Mode 指示 Claude 透過使用唯讀操作分析程式碼庫來建立計劃。

在會話期間打開 Plan Mode：使用 Shift+Tab 循環切換權限模式。

在 Plan Mode 中啟動新會話：

claude --permission-mode plan

恢復以前的對話

  • claude --continue 繼續當前目錄中最近的對話
  • claude --resume 打開對話選擇器或按名稱恢復

命名你的會話：

在啟動時使用 -n 命名會話：

claude -n auth-refactor

或在會話期間使用 /rename：

/rename auth-refactor

稍後按名稱恢復：

claude --resume auth-refactor

使用 Git worktrees 執行並行會話

當同時處理多個任務時，使用 --worktree（-w）旗標建立隔離的 worktree 並在其中啟動 Claude：

# 在名為 "feature-auth" 的 worktree 中啟動 Claude
claude --worktree feature-auth

# 在單獨的 worktree 中啟動另一個會話
claude --worktree bugfix-123

# 自動生成名稱
claude --worktree

Worktrees 在 <repo>/.claude/worktrees/<name> 建立。當你退出 worktree 會話時，worktree 及其分支會自動刪除（如果無更改）或提示你決定。

將 Claude 用作 Unix 風格的工具

管道進入、管道輸出：

cat build-error.txt | claude -p 'concisely explain the root cause of this build error' > output.txt

控制輸出格式：

  • 文字格式（預設）：--output-format text
  • JSON 格式：--output-format json
  • 串流 JSON 格式：--output-format stream-json
