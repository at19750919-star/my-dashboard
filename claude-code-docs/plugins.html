CHAPTER 05

Plugins 外掛系統

外掛建立與發佈、市集管理、Skills/Agents/Hooks/MCP 打包分發

建立外掛 Creating Plugins

外掛讓你能以自訂功能擴充 Claude Code，這些功能可在專案和團隊之間共享。本指南涵蓋如何使用 skills、agents、hooks 和 MCP servers 建立自己的外掛。

何時使用外掛與獨立配置 When to use plugins vs standalone configuration

Claude Code 支援兩種方式新增自訂 skills、agents 和 hooks：

方法 Skill 名稱 最適合
獨立 Standalone（.claude/ 目錄） /hello 個人工作流、專案專屬自訂、快速實驗
外掛 Plugin（含 .claude-plugin/plugin.json 的目錄） /plugin-name:hello 與團隊成員共享、發佈到社群、版本化發行、跨專案重用

在以下情況使用獨立配置

  • 你正為單一專案自訂 Claude Code
  • 配置是個人的，不需要共享
  • 你在打包 skills 或 hooks 前進行實驗
  • 你想要簡短的 skill 名稱，如 /hello 或 /deploy

在以下情況使用外掛

  • 你想與團隊或社群共享功能
  • 你需要在多個專案中使用相同的 skills/agents
  • 你想要版本控制和輕鬆更新擴充
  • 你透過市集分發
  • 你可以接受命名空間化的 skills，如 /my-plugin:hello（命名空間可防止外掛之間的衝突）
提示：從 .claude/ 中的獨立配置開始快速迭代，準備好共享時再轉換為外掛。

快速開始 Quick start

本快速開始將引導你建立帶有自訂 skill 的外掛。你將建立一個清單 manifest（定義外掛的配置檔）、新增一個 skill，並在本機測試。

建立你的第一個外掛 Create your first plugin

步驟 1：建立外掛目錄

每個外掛都位於自己的目錄中，包含清單和你的 skills、agents 或 hooks。現在建立一個：

mkdir my-first-plugin

步驟 2：建立外掛清單 Plugin Manifest

位於 .claude-plugin/plugin.json 的清單檔定義外掛的身份：名稱、描述和版本。Claude Code 使用此 metadata 中繼資料在外掛管理器中顯示你的外掛。

在外掛資料夾內建立 .claude-plugin 目錄：

mkdir my-first-plugin/.claude-plugin

然後建立 my-first-plugin/.claude-plugin/plugin.json：

{
  "name": "my-first-plugin",
  "description": "A greeting plugin to learn the basics",
  "version": "1.0.0",
  "author": {
    "name": "Your Name"
  }
}
欄位 用途
name 唯一識別碼和 skill 命名空間。Skills 以此為前綴（例如 /my-first-plugin:hello）。
description 在瀏覽或安裝外掛時於外掛管理器中顯示。
version 使用語義版本控制追蹤發行版本。
author 可選。有助於歸屬標示。

步驟 3：新增 Skill

Skills 位於 skills/ 目錄中。每個 skill 是一個包含 SKILL.md 檔案的資料夾。資料夾名稱成為 skill 名稱，以外掛的命名空間為前綴。

在外掛資料夾中建立 skill 目錄：

mkdir -p my-first-plugin/skills/hello

然後建立 my-first-plugin/skills/hello/SKILL.md：

---
description: Greet the user with a friendly message
disable-model-invocation: true
---

Greet the user warmly and ask how you can help them today.

步驟 4：測試你的外掛

使用 --plugin-dir 旗標執行 Claude Code 以載入你的外掛：

claude --plugin-dir ./my-first-plugin

Claude Code 啟動後，嘗試你的新 skill：

/my-first-plugin:hello

你會看到 Claude 用問候語回應。執行 /help 以查看你的 skill 列在外掛命名空間下。

注意：為什麼要命名空間？ 外掛 skills 總是命名空間化的（如 /greet:hello），以防止多個外掛有相同名稱的 skills 時發生衝突。要更改命名空間前綴，請更新 plugin.json 中的 name 欄位。

步驟 5：新增 Skill 參數

透過接受使用者輸入使你的 skill 動態化。$ARGUMENTS 佔位符擷取使用者在 skill 名稱後提供的任何文字。

更新你的 SKILL.md 檔案：

---
description: Greet the user with a personalized message
---

# Hello Skill

Greet the user named "$ARGUMENTS" warmly and ask how you can help them today. Make the greeting personal and encouraging.

執行 /reload-plugins 以取得變更，然後嘗試使用你的名字：

/my-first-plugin:hello Alex

外掛結構概覽 Plugin structure overview

外掛可以包含多個元件：自訂 agents、hooks、MCP servers 和 LSP servers。

警告：常見錯誤：不要將 commands/、agents/、skills/ 或 hooks/ 放在 .claude-plugin/ 目錄內。只有 plugin.json 應該在 .claude-plugin/ 裡。所有其他目錄必須在外掛根層級。
目錄 位置 用途
.claude-plugin/ 外掛根目錄 包含 plugin.json 清單（如果元件使用預設位置則可選）
commands/ 外掛根目錄 作為 Markdown 檔案的 Skills
agents/ 外掛根目錄 自訂 agent 定義
skills/ 外掛根目錄 帶有 SKILL.md 檔案的 Agent Skills
hooks/ 外掛根目錄 hooks.json 中的事件處理程式
.mcp.json 外掛根目錄 MCP server 配置
.lsp.json 外掛根目錄 用於程式碼智慧的 LSP server 配置
settings.json 外掛根目錄 啟用外掛時套用的預設設定

開發更複雜的外掛 Develop more complex plugins

向外掛新增 Skills Adding Skills to your plugin

外掛可以包含 Agent Skills 以擴充 Claude 的功能。Skills 是模型呼叫的：Claude 根據任務上下文自動使用它們。

在外掛根目錄新增 skills/ 目錄，其中包含帶有 SKILL.md 檔案的 Skill 資料夾：

my-plugin/
├── .claude-plugin/
│   └── plugin.json
└── skills/
    └── code-review/
        └── SKILL.md

每個 SKILL.md 需要包含 name 和 description 欄位的 frontmatter 前置資料，後面接說明：

---
name: code-review
description: Reviews code for best practices and potential issues. Use when reviewing code, checking PRs, or analyzing code quality.
---

When reviewing code, check for:
1. Code organization and structure
2. Error handling
3. Security concerns
4. Test coverage

安裝外掛後，執行 /reload-plugins 以載入 Skills。

向外掛新增 LSP servers Adding LSP servers to your plugin

提示：對於 TypeScript、Python 和 Rust 等常見語言，請從官方市集安裝預建的 LSP 外掛。僅當你需要支援尚未涵蓋的語言時，才建立自訂 LSP 外掛。

LSP（Language Server Protocol 語言伺服器協定）外掛為 Claude 提供即時程式碼智慧。如果你需要支援沒有官方 LSP 外掛的語言，可以向你的外掛新增 .lsp.json 檔案：

{
  "go": {
    "command": "gopls",
    "args": ["serve"],
    "extensionToLanguage": {
      ".go": "go"
    }
  }
}

安裝你的外掛的使用者必須在其機器上安裝語言伺服器二進位檔。

使用外掛提供預設設定 Shipping default settings with your plugin

外掛可在根目錄包含 settings.json 檔案，以在啟用外掛時套用預設配置。目前僅支援 agent 鍵。

設定 agent 會啟用外掛的自訂 agents 之一作為主執行緒，套用其系統提示、工具限制和模型。

{
  "agent": "security-reviewer"
}

此範例啟用在外掛的 agents/ 目錄中定義的 security-reviewer agent。

在本機測試外掛 Test your plugins locally

使用 --plugin-dir 旗標在開發期間測試外掛。這會直接載入你的外掛，無需安裝。

claude --plugin-dir ./my-plugin

當 --plugin-dir 外掛與已安裝的市集外掛同名時，本機副本在該工作階段中優先。這讓你可以測試已安裝外掛的變更，而無需先解除安裝。

當你對外掛進行變更時，執行 /reload-plugins 以取得更新，無需重新啟動。

提示：你可以透過多次指定旗標一次載入多個外掛： 
claude --plugin-dir ./plugin-one --plugin-dir ./plugin-two

除錯外掛問題 Debugging plugin issues

如果你的外掛不按預期運作：

  1. 檢查結構：確保你的目錄在外掛根目錄，而不是在 .claude-plugin/ 內
  2. 單獨測試元件：分別檢查每個 command、agent 和 hook
  3. 使用驗證和除錯工具：CLI 指令和疑難排解技術

將現有配置轉換為外掛 Convert existing configurations to plugins

如果你已經在 .claude/ 目錄中有 skills 或 hooks，可以將它們轉換為外掛，以便更輕鬆地共享和分發。

遷移步驟 Migration steps

步驟 1：建立外掛結構

mkdir -p my-plugin/.claude-plugin

在 my-plugin/.claude-plugin/plugin.json 建立清單檔：

{
  "name": "my-plugin",
  "description": "Migrated from standalone configuration",
  "version": "1.0.0"
}

步驟 2：複製現有檔案

# Copy commands
cp -r .claude/commands my-plugin/

# Copy agents (if any)
cp -r .claude/agents my-plugin/

# Copy skills (if any)
cp -r .claude/skills my-plugin/

步驟 3：遷移 Hooks

如果你在設定中有 hooks，請建立 hooks 目錄：

mkdir my-plugin/hooks

使用你的 hooks 配置建立 my-plugin/hooks/hooks.json。從 .claude/settings.json 或 settings.local.json 複製 hooks 物件，格式相同：

{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write|Edit",
        "hooks": [{ "type": "command", "command": "jq -r '.tool_input.file_path' | xargs npm run lint:fix" }]
      }
    ]
  }
}

步驟 4：測試已遷移的外掛

claude --plugin-dir ./my-plugin

測試每個元件：執行你的 commands、檢查 agents 是否出現在 /agents 中，並驗證 hooks 是否正確觸發。

遷移時的變化 What changes during migration

獨立 Standalone（.claude/） 外掛 Plugin
僅在一個專案中可用 可以透過市集共享
.claude/commands/ 中的檔案 plugin-name/commands/ 中的檔案
settings.json 中的 Hooks hooks/hooks.json 中的 Hooks
必須手動複製以共享 使用 /plugin install 安裝

外掛參考 Plugins Reference

Claude Code 外掛系統的完整技術參考，包括架構、CLI 指令和元件規格。

外掛元件參考 Plugin component reference

Skills

Plugins 向 Claude Code 新增 skills，建立可由你或 Claude 呼叫的 /name 快捷方式。

位置：外掛根目錄中的 skills/ 或 commands/ 目錄

檔案格式：Skills 是包含 SKILL.md 的目錄；commands 是簡單的 markdown 檔案

Skill 結構：

skills/
├── pdf-processor/
│   ├── SKILL.md
│   ├── reference.md (可選)
│   └── scripts/ (可選)
└── code-reviewer/
    └── SKILL.md

Agents

Plugins 可以為特定任務提供專門的 subagents 子代理，Claude 可以在適當時自動呼叫。

位置：外掛根目錄中的 agents/ 目錄

Agent 結構：

---
name: agent-name
description: 該 agent 的專長以及 Claude 應何時呼叫它
model: sonnet
effort: medium
maxTurns: 20
disallowedTools: Write, Edit
---

詳細的系統提示，描述 agent 的角色、專業知識和行為。

Plugin agents 支援 name、description、model、effort、maxTurns、tools、disallowedTools、skills、memory、background 和 isolation frontmatter 欄位。唯一有效的 isolation 值是 "worktree"。基於安全考量，plugin 提供的 agents 不支援 hooks、mcpServers 和 permissionMode。

Hooks

Plugins 可以提供事件處理程式，自動回應 Claude Code 事件。

位置：外掛根目錄中的 hooks/hooks.json，或在 plugin.json 中內嵌

Hook 配置：

{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write|Edit",
        "hooks": [
          {
            "type": "command",
            "command": "${CLAUDE_PLUGIN_ROOT}/scripts/format-code.sh"
          }
        ]
      }
    ]
  }
}

支援的事件：

事件 Event 觸發時機
SessionStart 工作階段開始或繼續時
UserPromptSubmit 你提交提示時，Claude 處理之前
PreToolUse 工具呼叫執行之前。可以阻止它
PermissionRequest 權限對話框出現時
PostToolUse 工具呼叫成功之後
PostToolUseFailure 工具呼叫失敗之後
Notification Claude Code 發送通知時
SubagentStart 子代理被產生時
SubagentStop 子代理完成時
TaskCreated 透過 TaskCreate 建立任務時
TaskCompleted 任務被標記為完成時
Stop Claude 完成回應時
StopFailure 因 API 錯誤結束回合時
FileChanged 監控的檔案在磁碟上變更時
WorktreeCreate 透過 --worktree 或 isolation: "worktree" 建立 worktree 時
WorktreeRemove worktree 被移除時
PreCompact 上下文壓縮之前
PostCompact 上下文壓縮完成之後
Elicitation MCP server 在工具呼叫期間請求使用者輸入時
ElicitationResult 使用者回應 MCP elicitation 後
SessionEnd 工作階段終止時

Hook 類型：

  • command：執行 shell 命令或指令碼
  • http：將事件 JSON 作為 POST 請求發送到 URL
  • prompt：使用 LLM 評估提示
  • agent：執行具有工具的 agentic 驗證器以完成複雜驗證任務

MCP servers

Plugins 可以綁定 Model Context Protocol (MCP) servers，將 Claude Code 與外部工具和服務連接。

位置：外掛根目錄中的 .mcp.json，或在 plugin.json 中內嵌

MCP server 配置：

{
  "mcpServers": {
    "plugin-database": {
      "command": "${CLAUDE_PLUGIN_ROOT}/servers/db-server",
      "args": ["--config", "${CLAUDE_PLUGIN_ROOT}/config.json"],
      "env": {
        "DB_PATH": "${CLAUDE_PLUGIN_ROOT}/data"
      }
    },
    "plugin-api-client": {
      "command": "npx",
      "args": ["@company/mcp-server", "--plugin-mode"],
      "cwd": "${CLAUDE_PLUGIN_ROOT}"
    }
  }
}

LSP servers

Plugins 可以提供 Language Server Protocol (LSP) servers，在處理程式碼庫時為 Claude 提供即時程式碼智慧。

位置：外掛根目錄中的 .lsp.json，或在 plugin.json 中內嵌

格式：將語言伺服器名稱對應到其配置的 JSON 配置

{
  "go": {
    "command": "gopls",
    "args": ["serve"],
    "extensionToLanguage": {
      ".go": "go"
    }
  }
}

必要欄位：

欄位 描述
command 要執行的 LSP 二進位檔（必須在 PATH 中）
extensionToLanguage 將檔案副檔名對應到語言識別碼

可選欄位：

欄位 描述
args LSP server 的命令列引數
transport 通訊傳輸方式：stdio（預設）或 socket
env 啟動 server 時要設定的環境變數
initializationOptions 在初始化期間傳遞給 server 的選項
settings 透過 workspace/didChangeConfiguration 傳遞的設定
workspaceFolder server 的工作區資料夾路徑
startupTimeout 等待 server 啟動的最長時間（毫秒）
shutdownTimeout 等待正常關閉的最長時間（毫秒）
restartOnCrash server 當機時是否自動重啟
maxRestarts 放棄前的最大重啟嘗試次數
警告：你必須另外安裝語言伺服器二進位檔。 LSP 外掛配置 Claude Code 如何連接到語言伺服器，但它們不包括伺服器本身。如果在 /plugin Errors（錯誤）分頁中看到 Executable not found in $PATH，請為你的語言安裝所需的二進位檔。

外掛安裝範圍 Plugin installation scopes

安裝外掛時，你選擇一個範圍 scope，決定外掛的可用位置以及誰可以使用：

範圍 Scope 設定檔 使用案例
user ~/.claude/settings.json 在所有專案中可用的個人外掛（預設）
project .claude/settings.json 透過版本控制共享的團隊外掛
local .claude/settings.local.json 專案專屬的外掛，gitignored
managed 託管設定 託管外掛（唯讀，僅更新）

外掛清單結構描述 Plugin manifest schema

.claude-plugin/plugin.json 檔案定義外掛的中繼資料和配置。

完整結構描述：

{
  "name": "plugin-name",
  "version": "1.2.0",
  "description": "Brief plugin description",
  "author": {
    "name": "Author Name",
    "email": "author@example.com",
    "url": "https://github.com/author"
  },
  "homepage": "https://docs.example.com/plugin",
  "repository": "https://github.com/author/plugin",
  "license": "MIT",
  "keywords": ["keyword1", "keyword2"],
  "commands": ["./custom/commands/special.md"],
  "agents": "./custom/agents/",
  "skills": "./custom/skills/",
  "hooks": "./config/hooks.json",
  "mcpServers": "./mcp-config.json",
  "outputStyles": "./styles/",
  "lspServers": "./.lsp.json"
}

必要欄位：

如果包含清單，name 是唯一必要欄位。

欄位 型別 描述 範例
name string 唯一識別碼（kebab-case 短橫線命名，無空格） "deployment-tools"

中繼資料欄位：

欄位 型別 描述 範例
version string 語義版本 "2.1.0"
description string 外掛用途的簡要說明 "Deployment automation tools"
author object 作者資訊 {"name": "Dev Team", "email": "dev@company.com"}
homepage string 文件 URL "https://docs.example.com"
repository string 原始碼 URL "https://github.com/user/plugin"
license string 授權識別碼 "MIT"、"Apache-2.0"
keywords array 探索標籤 ["deployment", "ci-cd"]

使用者配置 User configuration

userConfig 欄位宣告 Claude Code 在啟用外掛時提示使用者的值。

{
  "userConfig": {
    "api_endpoint": {
      "description": "您團隊的 API 端點",
      "sensitive": false
    },
    "api_token": {
      "description": "API 身份驗證權杖",
      "sensitive": true
    }
  }
}

鍵必須是有效的識別碼。每個值都可在 MCP 和 LSP server 配置、hook 命令以及（僅對非敏感值）skill 和 agent 內容中作為 ${user_config.KEY} 進行替換。

環境變數 Environment variables

Claude Code 提供兩個變數用於引用外掛路徑：

${CLAUDE_PLUGIN_ROOT}：外掛安裝目錄的絕對路徑。使用此路徑引用與外掛綁定的指令碼、二進位檔和配置檔。

${CLAUDE_PLUGIN_DATA}：用於外掛狀態的持久目錄，在更新後保留。使用此目錄存放已安裝的依賴項、產生的程式碼、快取以及任何應在外掛版本之間保留的檔案。

版本管理 Version management

遵循語義版本控制 Semantic Versioning 進行外掛發行：

{
  "name": "my-plugin",
  "version": "2.1.0"
}

版本格式：MAJOR.MINOR.PATCH

  • MAJOR：破壞性變更（不相容的 API 變更）
  • MINOR：新功能（向後相容的新增）
  • PATCH：錯誤修復（向後相容的修復）

最佳實務：

  • 從 1.0.0 開始進行第一個穩定版本
  • 在分發變更之前更新 plugin.json 中的版本
  • 在 CHANGELOG.md 檔案中記錄變更
  • 使用預發行版本如 2.0.0-beta.1 進行測試
警告：Claude Code 使用版本來決定是否更新你的外掛。如果你變更了外掛的程式碼但沒有在 plugin.json 中提升版本，現有使用者因為快取而看不到你的變更。

除錯和開發工具 Debugging and development tools

除錯指令：使用 claude --debug 查看外掛載入詳情。這會顯示：

  • 正在載入哪些外掛
  • 外掛清單中的任何錯誤
  • 命令、agent 和 hook 註冊
  • MCP server 初始化

常見問題：

問題 原因 解決方案
外掛未載入 無效的 plugin.json 執行 claude plugin validate 或 /plugin validate 檢查語法和結構描述錯誤
命令未出現 目錄結構錯誤 確保 commands/ 在根目錄，而不是在 .claude-plugin/ 中
Hooks 未觸發 指令碼不可執行 執行 chmod +x script.sh
MCP server 失敗 缺少 ${CLAUDE_PLUGIN_ROOT} 對所有外掛路徑使用變數
路徑錯誤 使用了絕對路徑 所有路徑必須是相對的，並以 ./ 開頭
LSP Executable not found in $PATH 語言伺服器未安裝 安裝二進位檔（例如 npm install -g typescript-language-server typescript）

建立和分發外掛市集 Create and Distribute Plugin Marketplaces

**plugin marketplace 外掛市集**是一個目錄，讓你能將外掛分發給他人。Marketplace 提供集中式探索、版本追蹤、自動更新以及對多種來源類型（git 儲存庫、本機路徑等）的支援。

概述 Overview

建立和分發市集涉及：

  1. 建立外掛：使用 commands、agents、hooks、MCP servers 或 LSP servers 建構一個或多個外掛
  2. 建立市集檔案：定義一個 marketplace.json，列出你的外掛及其位置
  3. 託管市集：推送到 GitHub、GitLab 或其他 git 主機
  4. 與使用者共享：使用者使用 /plugin marketplace add 新增你的市集並安裝單個外掛

演練：建立本機市集 Walkthrough: create a local marketplace

此範例建立一個包含一個外掛的市集：一個用於程式碼審查的 /quality-review skill。

步驟 1：建立目錄結構

mkdir -p my-marketplace/.claude-plugin
mkdir -p my-marketplace/plugins/quality-review-plugin/.claude-plugin
mkdir -p my-marketplace/plugins/quality-review-plugin/skills/quality-review

步驟 2：建立 Skill

---
description: Review code for bugs, security, and performance
disable-model-invocation: true
---

Review the code I've selected or the recent changes for:
- Potential bugs or edge cases
- Security concerns
- Performance issues
- Readability improvements

Be concise and actionable.

步驟 3：建立 Plugin Manifest 外掛清單

{
  "name": "quality-review-plugin",
  "description": "Adds a /quality-review skill for quick code reviews",
  "version": "1.0.0"
}

步驟 4：建立市集檔案

建立列出你的外掛的市集目錄：

{
  "name": "my-plugins",
  "owner": {
    "name": "Your Name"
  },
  "plugins": [
    {
      "name": "quality-review-plugin",
      "source": "./plugins/quality-review-plugin",
      "description": "Adds a /quality-review skill for quick code reviews"
    }
  ]
}

步驟 5：新增和安裝

/plugin marketplace add ./my-marketplace
/plugin install quality-review-plugin@my-plugins

步驟 6：試試看

/quality-review

建立市集檔案 Create the marketplace file

在你的儲存庫根目錄中建立 .claude-plugin/marketplace.json。此檔案定義市集的名稱、擁有者資訊以及包含來源的外掛清單。

{
  "name": "company-tools",
  "owner": {
    "name": "DevTools Team",
    "email": "devtools@example.com"
  },
  "plugins": [
    {
      "name": "code-formatter",
      "source": "./plugins/formatter",
      "description": "Automatic code formatting on save",
      "version": "2.1.0",
      "author": {
        "name": "DevTools Team"
      }
    },
    {
      "name": "deployment-tools",
      "source": {
        "source": "github",
        "repo": "company/deploy-plugin"
      },
      "description": "Deployment automation tools"
    }
  ]
}

市集結構描述 Marketplace schema

必要欄位：

欄位 型別 描述 範例
name string 市集識別碼（kebab-case 短橫線命名，無空格） "acme-tools"
owner object 市集維護者資訊
plugins array 可用外掛清單

外掛來源 Plugin sources

外掛來源告訴 Claude Code 在你的市集中列出的每個外掛從哪裡取得。

來源 型別 欄位 注意事項
相對路徑 string（例如 "./my-plugin"） -- 市集 repo 中的本機目錄。必須以 ./ 開頭
github object repo、ref?、sha?
url object url、ref?、sha? Git URL 來源
git-subdir object url、path、ref?、sha? git repo 中的子目錄
npm object package、version?、registry? 透過 npm install 安裝

GitHub 儲存庫 GitHub repository

{
  "name": "github-plugin",
  "source": {
    "source": "github",
    "repo": "owner/plugin-repo",
    "ref": "v2.0.0",
    "sha": "a1b2c3d4e5f6a7b8c9d0e1f2a3b4c5d6e7f8a9b0"
  }
}

Git 儲存庫 Git repository

{
  "name": "git-plugin",
  "source": {
    "source": "url",
    "url": "https://gitlab.com/team/plugin.git",
    "ref": "main",
    "sha": "a1b2c3d4e5f6a7b8c9d0e1f2a3b4c5d6e7f8a9b0"
  }
}

npm 套件 npm package

{
  "name": "my-npm-plugin",
  "source": {
    "source": "npm",
    "package": "@acme/claude-plugin",
    "version": "2.1.0",
    "registry": "https://npm.example.com"
  }
}

託管和分發市集 Host and distribute marketplaces

在 GitHub 上託管（推薦） Host on GitHub (recommended)

GitHub 提供最簡單的分發方法：

  1. 建立儲存庫：為你的市集設定一個新儲存庫
  2. 新增市集檔案：使用你的外掛定義建立 .claude-plugin/marketplace.json
  3. 與團隊共享：使用者使用 /plugin marketplace add owner/repo 新增你的市集

在其他 git 服務上託管 Host on other git services

任何 git 託管服務都可以運作，例如 GitLab、Bitbucket 和自架伺服器：

/plugin marketplace add https://gitlab.com/company/plugins.git

私有儲存庫 Private repositories

Claude Code 支援從私有儲存庫安裝外掛。對於手動安裝和更新，Claude Code 使用你現有的 git 憑證助手。

後台自動更新在啟動時執行，不使用憑證助手。要為私有市集啟用自動更新，請在你的環境中設定適當的身份驗證權杖：

供應商 環境變數 注意事項
GitHub GITHUB_TOKEN 或 GH_TOKEN 個人存取權杖或 GitHub App 權杖
GitLab GITLAB_TOKEN 或 GL_TOKEN 個人存取權杖或專案權杖
Bitbucket BITBUCKET_TOKEN 應用程式密碼或儲存庫存取權杖

在你的 shell 配置中設定權杖：

export GITHUB_TOKEN=ghp_xxxxxxxxxxxxxxxxxxxx

為團隊要求市集 Require marketplaces for your team

你可以配置儲存庫，以便當團隊成員信任專案資料夾時自動被提示安裝你的市集。將你的市集新增到 .claude/settings.json：

{
  "extraKnownMarketplaces": {
    "company-tools": {
      "source": {
        "source": "github",
        "repo": "your-org/claude-plugins"
      }
    }
  }
}

你也可以指定預設應啟用哪些外掛：

{
  "enabledPlugins": {
    "code-formatter@company-tools": true,
    "deployment-tools@company-tools": true
  }
}

託管市集限制 Managed marketplace restrictions

對於需要嚴格控制外掛來源的組織，管理員可以使用託管設定中的 strictKnownMarketplaces 設定限制使用者允許新增的外掛市集。

當在託管設定中配置 strictKnownMarketplaces 時，限制行為取決於值：

值 行為
未定義（預設） 無限制。使用者可以新增任何市集
空陣列 [] 完全鎖定。使用者無法新增任何新市集
來源清單 使用者只能新增與允許清單完全匹配的市集

常見配置 Common configurations

停用所有市集新增：

{
  "strictKnownMarketplaces": []
}

僅允許特定市集：

{
  "strictKnownMarketplaces": [
    {
      "source": "github",
      "repo": "acme-corp/approved-plugins"
    },
    {
      "source": "github",
      "repo": "acme-corp/security-tools",
      "ref": "v2.0"
    },
    {
      "source": "url",
      "url": "https://plugins.example.com/marketplace.json"
    }
  ]
}

驗證和測試 Validation and testing

在共享前測試你的市集。

驗證你的市集 JSON 語法：

claude plugin validate .

或從 Claude Code 內：

/plugin validate .

新增市集進行測試：

/plugin marketplace add ./path/to/marketplace

安裝測試外掛以驗證一切正常：

/plugin install test-plugin@marketplace-name
